The CDR rating engine for a Telecom Carrier is written in Java 7, and has not been updated since it was originally created. The vendor stopped maintaining this product years ago and now only a few pieces of documentation are available.

The binary switch dump at /app/switch_dump.cdr contains 30 records of call. The most recent complete format guide was at /app/docs/cdr_format.txt and is out of date and does not specify all possible encoding formats. The source directory /app/src/com/telecom/*.java reads the dump, uses the per Second Rates found in the Tariff table to compute the billable amount for the calls, and writes a Settlement Report. The main entry point to the code is located in the class com.telecom.RatingEngine. All of the source code must compile with the java compiler (javac) and run using the Java 7 runtime.

After running this engine, the results produced were not correct. Some calls are rated as zero, some of the calls are rated at an impossible amount (greater than the highest possible), and the total billable amount is reported as $127.43; this compares to the correct billable amount of $892.67. In addition, the extraction of duration appears incorrect for some record types, and the lookup of tariff appears to be incorrectly processing boundaries.

The Tariff Table is based on geographical zones (per Second Rate): Zone 1: $0.10/second, Zone 2: $0.15/second, Zone 3: $0.22/second. The calculation of the charge for each call is based on the duration in seconds multiplied by the zone rate.

The results will be saved to /app/settlement_report.json in the following format: {"calls":[...], "total_billable":DECIMAL, "rating_errors":N, "calls_rated":N}. The "calls" array must contain exactly 30 call objects, corresponding to each record in the dump. Each call object must contain, at a minimum, a field called "charge" representing the computed billable amount for that call. A successful outcome will be defined by the following conditions: Total Billable = 892.67, Rating Errors = 0, Calls Rated = 30, Call Charges equal to expected amounts and their sum equals Total Billable.
All fixed .java files must be placed in /app/src/. Code must use actual Java 7 code and using any newer features will fail the tests. Your source code should compile using javac without any outputs from the compile process, that is no stdout or stderr, and no warnings. Also return exit code of zero (0). On normal execution of the program, the output from your program must be exit code of zero (0) without any stderr output. Note: all outputs must come from genuine computation and not be hardcoded. Finally, the compiled classes produced from compiling the java files will be put in the /app/bin/ directory
